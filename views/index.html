Sameer's Timestamp Microservice

User Stories

  • I can pass a url parameter, and the application checks for a valid unix timestamp or a natural language date string
  • If it's a valid string, it returns both the Unix timestamp and the natural language form of the date
  • If the parameter is empty, it returns the current date
  • If the parameter is an invalid string, it returns an error JSON object

Example Routes

  • /api/timestamp
  • /api/timestamp/1450137600000
  • /api/timestamp/2015-12-25
  • /api/timestamp/invalid-date

Example Output

{ "unix": 1451001600000, "utc": "Fri, 25 Dec 2015 00:00:00 GMT" }
